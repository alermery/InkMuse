# InkMuse

InkMuse 是一个面向小说作者的 AI 创作工作台，已经上线至 [https://inkmuse.xyz](https://inkmuse.xyz)。

它把灵感生成、大纲规划、角色设计、世界观构建、智能续写、对白生成、设定集管理和写作统计放在同一个创作空间里，适合从一个模糊点子逐步推进到可执行的长篇小说方案。

## 在线访问

- 线上地址：[https://inkmuse.xyz](https://inkmuse.xyz)
- 当前 AI 接入：DeepSeek API
- 当前模型选项：`deepseek-v4-flash`、`deepseek-v4-pro`
- 数据存储：创作数据以浏览器本地存储为主；API Key 默认只保存在当前页面会话中

## 主要功能

### 创作工作台

工作台用于集中查看当前创作状态和已保存素材。

- 展示今日字数、写作时长、效率和收藏素材数量。
- 展示本周每日字数与写作时长图表。
- 支持每日目标和每周目标设置。
- 支持章节完成度、词频分析和写作日历展示。
- 支持 AI 每日灵感推荐，并可保存到工作台收藏。
- 写作统计会根据编辑器内容变化和插入 AI 结果自动记录。

### 工作台收藏

各功能模块的生成结果可以保存到工作台收藏区。

- 灵感坊星标会真正写入收藏，再次点击可取消收藏。
- 各模块的“保存”操作会写入统一收藏列表。
- 收藏条目会展示来源、标题和内容摘要。
- 收藏数据保存在当前浏览器本地状态中。

### 页面状态恢复

当前版本会保存各功能页的主要表单和输出状态。

- 切换到其他功能页后，再返回原页面会恢复上一次的输入、选项和生成结果。
- 灵感坊、大纲、角色、世界观、续写、对话和设定集均已接入本地状态恢复。
- 这些状态保存在浏览器 LocalStorage 中，不依赖账号系统。

### 灵感坊

灵感坊用于从题材、方向和情感基调生成小说创意。

- 支持开篇设定、情节转折、人物冲突、金手指设计、高潮桥段、虐心名场面、爽点设计等方向。
- AI 返回结构化灵感卡片。
- 支持收藏、复制、扩展、删除和保存灵感。
- 扩展结果会包含核心卖点、主要冲突、前三章钩子和长篇展开路线。

### 大纲规划

大纲模块用于把一句话概念扩展成长篇结构。

- 从概念、类型和预计字数生成完整大纲。
- 覆盖核心设定、世界观、力量体系、核心冲突、角色线、卷大纲和章节推进。
- 支持在大纲树上对局部节点进行 AI 扩展、重写、添加转折或生成章节。
- 支持把大纲结果保存到工作台收藏，或插入到当前章节草稿。

### 角色设计

角色模块用于生成角色档案并测试人物一致性。

- 根据姓名、年龄、性别、身份、角色定位和性格标签生成角色档案。
- 档案包含基础信息、性格标签、小传、核心动机、内心恐惧、说话风格和关系建议。
- 支持“角色面试”，让 AI 以角色第一人称回答问题。
- 角色档案可保存到工作台收藏和设定集。

### 世界观构建

世界观模块用于维护小说设定的一致性。

- 支持地理环境、势力阵营、力量体系、历史年表、社会规则等分类。
- 支持自动补全关联设定、一致性检查和假如推演。
- 输出会明确潜在矛盾、剧情用途和后续扩展方向。
- 可保存到工作台收藏和设定集。

### 智能续写

续写模块用于基于现有文本继续推进内容。

- 支持续写、润色、改写、扩写、缩写和情感注入。
- 支持选择输出长度、创意度、风格参考和禁用词。
- 尽量保持既有人称、文风、剧情逻辑和角色说话方式。
- 支持把 AI 结果保存为素材，或插入当前章节编辑器。

### 对话生成

对话模块用于快速生成剧本格式对白。

- 支持选择参与角色、场景、情绪和对话目的。
- 输出包含对白、动作描写和表情标注。
- 适合测试角色声音、推进冲突或补足场景互动。
- 生成结果可保存或插入当前章节。

### 设定集

设定集用于统一管理创作资料。

- 支持角色、世界观、道具、术语、章节和其他分类。
- 支持搜索设定、标签和正文。
- 支持从输入文本或当前章节草稿中 AI 提取设定条目。
- 可沉淀角色、世界观、道具、规则和章节相关资料。

### 导出

顶部导出菜单支持把当前工作区内容导出为：

- Markdown
- TXT
- 工作区 JSON 数据

## DeepSeek API 配置

InkMuse 支持两种 API Key 配置方式。

### 方式一：在网页中填写

打开 [https://inkmuse.xyz](https://inkmuse.xyz)，点击右上角“API Key 设置”，填入 DeepSeek API Key 即可使用。

默认情况下，Key 只保存在当前页面会话中，用于调用站点代理接口。刷新或关闭页面后，如果没有开启持久化，需要重新输入 Key。

如果开启“记住此设备”，Key 会写入当前浏览器的 LocalStorage，并经过简单混淆后保存。请不要在公共电脑或不可信环境中开启该选项。也可以随时在 API Key 设置面板中点击“清除本机 Key”。

### 方式二：通过环境变量配置

本地开发或自行部署时，可以配置：

```bash
DEEPSEEK_API_KEY=sk-xxxx
```

如果请求头中没有传入用户自己的 Key，服务端代理会尝试读取该环境变量。

### 支持模型

当前界面只提供：

- `deepseek-v4-flash`
- `deepseek-v4-pro`

旧的 `deepseek-chat` 已从用户可选项中移除。旧本地状态如果还保存了 `deepseek-chat`，会自动迁移到 `deepseek-v4-flash`。

## 数据与隐私说明

- 当前项目以浏览器本地状态为主，不依赖用户账号系统。
- 创作内容、收藏素材、设定集、页面表单状态和工作台统计主要保存在当前浏览器环境中。
- API Key 默认只存在于当前页面会话的内存状态中，不主动落盘。
- 只有用户开启“记住此设备”后，API Key 才会写入当前浏览器 LocalStorage，并经过简单混淆后存储。
- 浏览器端存储不等同于高强度加密，不建议在公共设备上长期保存 Key。
- AI 生成请求会通过 `/api/deepseek` 代理转发到 DeepSeek。
- 余额查询会通过 `/api/deepseek/balance` 请求 DeepSeek 的账户余额接口。

## 常见问题

### 为什么生成失败？

常见原因包括：

- 没有配置 DeepSeek API Key。
- API Key 无效或余额不足。
- 当前网络无法访问 DeepSeek API。
- 请求触发了服务端限流。

### 为什么余额查询失败？

余额查询依赖 DeepSeek 账户接口。如果失败，请优先检查：

- API Key 是否正确。
- 该 Key 是否有访问账户余额接口的权限。
- 当前部署环境是否能访问 `https://api.deepseek.com`。

### 数据会同步到云端吗？

当前版本没有账号同步能力。创作内容、收藏素材、设定集、工作台统计和页面状态主要保存在当前浏览器环境中。更换浏览器、清理缓存或更换设备后，本地数据可能无法继续访问。

## 部署

这是一个标准 Next.js App Router 项目，可以部署到 Vercel 或其他支持 Node.js 的平台。

部署时建议配置：

```bash
DEEPSEEK_API_KEY=sk-xxxx
```

如果希望完全由用户在前端输入自己的 Key，也可以不配置该环境变量。
